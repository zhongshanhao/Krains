代理模式--静态代理和动态代理

Krains 2020-12-24

静态代理，就是自己手写的代理类，而动态代理可以在程序运行期间根据需要动态创建代理类及其实例来完成具体的功能。

根据代理的创建时期，代理模式分为静态代理和动态代理。

  • 静态：由程序员创建代理类或特定工具自动生成源代码再对其编译，在程序运行前代理类的 .class 文件就已经存在了。
  • 动态：在程序运行时，运用反射机制动态创建而成

# 静态代理

静态代理的实现模式一般是：首先创建一个接口（JDK代理都是面向接口的），然后创建具体实现类来实现这个接口，然后再创建一个代理类同样实现这个接口。

// 被代理对象
class Son implements IPerson{

    public void findTeacher() {
        System.out.println("儿子要老师辅导功课");
    }
}

// 代理对象，能够增强被代理对象的功能
class Father implements IPerson{
    IPerson son;

    public Father(Son son){
        this.son = son;
    }

    public void findTeacher() {
        System.out.println("找老师");
        son.findTeacher();
        System.out.println("开始学习");
    }
}

public class StaticProxy {
    
    public static void main(String[] args) {
        // 使用代理对象
        Father father = new Father(new Son());
        father.findTeacher();
    }
}

静态代理的缺点

  • 代理类也要实现接口，一旦接口改变，相应的代理类也要做出改变
  • 如果很多业务类需要代理，那么就需要对每一个类添加一个代理类，这样会很繁琐

动态代理可以解决静态代理的缺点

# 动态代理

# JDK实现

使用 jdk Proxy类实现动态代理

需要实现InvocationHandler接口，重写invoke方法，该方法有三个参数

  • proxy：生成的代理对象
  • method：方法
  • args：方法参数
public interface InvocationHandler {
	public Object invoke(Object proxy, Method method, Object[] args);
}

通过Proxy的newProxyInstance方法获得代理对象，该方法签名如下

static Object newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h )
1

三个参数为

  • loader：被代理类的类加载器
  • interfaces：被代理对象实现的接口信息
  • h：事件处理器，调用被代理对象的接口方法时，会触发事件处理器的方法，也就是会调用h的invoke方法
// 使用jdk动态代理，传入被代理的对象，返回代理对象
class JdkFuDao implements InvocationHandler{

    private IPerson target;
    
    public IPerson getInstance(IPerson target){
        this.target = target;
        Class<?> clazz = target.getClass();
        return (IPerson) Proxy.newProxyInstance(clazz.getClassLoader(), clazz.getInterfaces(), this);
    }
    
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("调用方法前");
        Object ret = method.invoke(target, args);
        System.out.println("调用方法后");
        return ret;
    }
}

public class DynamicProxy {
    public static void main(String[] args) {
        JdkFuDao jdkFuDao = new JdkFuDao();
        IPerson instance = jdkFuDao.getInstance(new Son());
        instance.findTeacher();
    }

我们知道，在getProxyClass0()方法中通过反射解析被代理类，动态地生成了一个代理类，这个代理类实现了IPerson接口，继承了Proxy，Proxy接受一个InvocationHandler参数的构造方法，然后在实现了接口的方法中调用相应的处理方法，完成代理。

Java动态代理为什么只能够对实现接口的类进行代理？

newProxyInstance()返回的是Object对象，该代理对象继承了Proxy和实现了IPerson，如果要使用该代理类的话必须向下转型，用接口去强转，然后才能调用相应的方法。如果不是实现接口，那么用Proxy向下转型，但是Proxy没有findTeacher方法，所以不能执行，而 Java 不允许多继承，因此，代理类必须和被代理类实现相同的接口，这样可以充分发挥多态的特性。

如果要对一个类进行代理，而这个类没有实现接口，需要使用CGLib技术去实现代理。

System.setProperty(DebuggingClassWriter.DEBUG_LOCATION_PROPERTY,

        return result;

// 获取代理类实例，代理类是Son的子类，因此可以向上转型

最后我们总结一下 JDK 动态代理和Gglib动态代理的区别：

  • JDK动态代理是实现了被代理对象的接口，Cglib是继承了被代理对象。
  • JDK和Cglib都是在运行期生成字节码，JDK是直接写Class字节码，Cglib使用ASM框架写Class字节码，Cglib代理实现更复杂，生成代理类比JDK效率低。
  • JDK调用代理方法，是通过反射机制调用，Cglib是通过FastClass机制直接调用方法，Cglib执行效率更高。
  • 静态代理
  • 动态代理
